# ADR 0012：默认使用中文展示并保留协议原值

- 状态：已接受
- 日期：2026-07-10

## 背景

AuditCanvas 的主要使用者需要中文界面、中文命令说明和中文审计报告，但审计运行同时也是可交换、可验证的机器数据。直接翻译枚举值、规则 ID、文件路径或原始证据，会破坏既有 Schema、稳定 ID、外部工具兼容性和证据真实性。

## 决策

- Web、CLI 帮助、Markdown/HTML 报告、README 和 Codex 插件提示默认使用简体中文。
- Web 保留英语切换，并把用户选择持久化到浏览器本地存储。
- CLI 通过 `AUDIT_CANVAS_LOCALE=en` 或 `--locale en` 提供英语输出。
- JSON 导出继续使用协议定义的英文枚举和字段名，不做本地化。
- 状态、严重程度、分类和内置规则文案只在展示层本地化。
- 文件路径、行号、规则 ID、运行 ID、Git 引用和原始证据保持原值。
- 第三方分析器或人工录入的标题与解释保持原值，避免把未经确认的内容误译为审计事实。

## 后果

- 中文用户无需配置即可获得完整中文体验。
- 英语用户和自动化集成仍可使用稳定的命令、Schema 与 JSON 数据。
- 新增协议枚举时，Web 和报告导出器必须同步补齐中英文展示标签。
- 测试需要同时覆盖中文默认路径、英语回退路径和证据不变性。
